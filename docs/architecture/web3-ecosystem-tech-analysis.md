| --------------------- | --------------------------------------------------- | ------------------ |

### 3.3 完整审计流程

```
开发阶段 → 静态分析（Slither）
        → 符号执行（Mythril）
        → AI辅助审计
        → 发现漏洞 → 修复
        → 专业人工审计
        → 主网部署 → 实时监控
```

### 3.4 常见漏洞类型

1. **重入攻击（Reentrancy）** - 外部调用前状态更新
2. **整数溢出/下溢** - 使用SafeMath或Solidity 0.8+
3. **访问控制缺失** - 权限检查正确实现
4. **闪贷攻击** - 价格预言机保护
5. **代理合约风险** - 存储布局冲突检查

## 5. DEX流动性池解决方案

### 5.1 AMM模型类型

| 模型类型       | 公式       | 代表项目   | 特点                |
| -------------- | ---------- | ---------- | ------------------- |
| **恒定乘积**   | x × y = k  | Uniswap V2 | 简单、通用          |
| **集中流动性** | 多tick区间 | Uniswap V3 | 资本效率提升10-100x |
| **稳定币**     | 混合曲线   | Curve      | 低滑点、高稳定      |
| **动态费用**   | 可配置钩子 | Uniswap V4 | 高度可定制          |

### 5.2 Uniswap V3架构详解

#### 核心合约

| 合约                 | 职责     | 主要功能             |
| -------------------- | -------- | -------------------- |
| **UniswapV3Factory** | 工厂合约 | 创建交易池、管理费率 |
| **UniswapV3Pool**    | 交易池   | 流动性管理、代币交换 |
| **Periphery**        | 外围合约 | 用户交互、路由、NFT  |

#### 核心接口

```solidity
// 交易池核心操作
interface IUniswapV3PoolActions {
    initialize(sqrtPriceX96)                  // 初始化价格
    mint(recipient, tickLower, tickUpper, amount)  // 添加流动性
    burn(tickLower, tickUpper, amount)         // 移除流动性
    collect(recipient, tickLower, tickUpper)  // 提取费用
    swap(recipient, zeroForOne, amountSpecified, sqrtPriceLimitX96)  // 代币交换
}
```

#### 关键参数

| 参数                | 含义                    | 作用           |
| ------------------- | ----------------------- | -------------- |
| sqrtPriceX96        | Q64.96格式的价格平方根  | 精确价格表示   |
| tickLower/tickUpper | 流动性区间上下界        | 集中流动性范围 |
| fee                 | 费率（0.05%, 0.3%, 1%） | 不同风险池选择 |
| tickSpacing         | 初始化tick间隔          | 与费率关联     |
