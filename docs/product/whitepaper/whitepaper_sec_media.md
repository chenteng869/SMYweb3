## 3.4 自媒体中心模块（全新模块）

### 3.4.0 模块概述

自媒体中心是太初国链APP 3.0版本的核心新增模块，定位为跨境电商卖家的"内容中台"。该模块将内容创作、多平台分发、数据追踪、AI辅助、收益激励五大能力整合为统一闭环，帮助卖家以最低运营成本实现品牌内容在多平台的规模化曝光。

**模块核心价值主张：**

| 价值维度 | 传统方式痛点 | 自媒体中心解决方案 |
|---------|------------|------------------|
| 分发效率 | 需登录6+平台逐一发布，耗时30分钟/篇 | 一键勾选9大平台，3秒完成分发 |
| 内容质量 | 非专业写手，文案转化效率低 | GPT-4o多平台适配文案，平台化风格生成 |
| 数据追踪 | 各平台数据分散，无法横向对比 | 统一数据看板，跨平台ROI一目了然 |
| 创作激励 | 纯成本支出，无正向反馈 | 发布即赚DVC，内容贡献直接转化为链上资产 |
| 等级权益 | 所有用户功能同质化 | DLC等级越高，解锁平台越多、奖励越高、工具越强 |

#### 3.4.2.1 页面定位

内容发布页是自媒体中心的核心生产工具，覆盖内容从创建到分发的完整流程。页面采用"纵向单流"布局，按照内容创作的逻辑顺序自上而下排列各功能区块，确保用户在单次滚动中完成全部操作。

- **渐进式披露**：基础功能默认展示，高级功能（定时发布、平台自定义）折叠收起
- **智能默认**：根据用户DLC等级和历史行为自动填充推荐选项
- **实时校验**：标题字数、标签数量、平台选择等实时反馈合规性
- **防误触机制**：发布按钮二次确认，定时发布日期合理性校验

### 3.4.3 数据看板页

#### 3.4.3.1 页面定位

数据看板页是自媒体中心的"数据大脑"，为卖家提供跨平台内容表现的统一视图。通过聚合多平台数据并进行标准化清洗，看板消除了平台间数据口径差异，让用户能够真实评估内容ROI并指导后续创作方向。

**数据看板设计原则：**
- **指标精选**：只展示与业务目标强相关的核心指标，避免数据过载
- **趋势导向**：所有数值指标均附同比/环比趋势，强调变化而非绝对值
- **下钻能力**：汇总数据可点击下钻至明细，明细可下钻至单条内容
- **时效感知**：数据 freshness 明确标注，超过阈值提示重新同步

| 互动率 | (点赞+评论+转发) / 播放 x 100% | 加权平均 | 计算值 | 实时 |
| 粉丝净增 | 本期粉丝 - 上期粉丝 | 差值 | 计算值 | 实时 |
| 内容发布数 | 指定时间范围内发布的内容条数 | COUNT(内容) | CMS数据库 | 实时 |

#### 3.4.4.1 联动机制设计

#### 3.4.4.3 DVC奖励计算公式

```python
# DVC奖励计算逻辑（伪代码）
def calculate_content_reward(dlc_level: int, platforms: list, content_type: str) -> dict:
    """
    计算内容发布后的DVC奖励
    
    参数:
        dlc_level: 用户DLC等级 (1-9)
        platforms: 分发平台列表，如 ["douyin", "wechannel", "tiktok"]
        content_type: 内容类型 ("video" | "article" | "live" | "image_text")
    
    返回:
        {
            "base_reward": int,      # 基础奖励
            "platform_bonus": int,   # 平台数量加成
            "dlc_multiplier": float, # DLC等级加成系数
            "quality_bonus": int,    # 质量加成（基于AI评分）
            "total_reward": int,     # 总奖励
            "breakdown": dict        # 明细
        }
    """
    
    # 1. 基础奖励（根据内容类型）
    base_rewards = {
        "video": 200,      # 视频创作成本高，奖励最高
        "live": 150,       # 直播奖励
        "article": 120,    # 文章奖励
        "image_text": 100  # 图文奖励
    }
    base_reward = base_rewards.get(content_type, 100)
    
    # 2. 平台数量加成（每多一个平台+20%）
    platform_count = len(platforms)
    platform_bonus_rate = 1.0 + (platform_count - 1) * 0.2
    
    # 3. DLC等级加成系数
    dlc_multipliers = {
        1: 1.0, 2: 1.0,
        3: 1.2, 4: 1.2,
        5: 1.5, 6: 1.5,
        7: 2.0, 8: 2.0,
        9: 3.0
    }
    dlc_multiplier = dlc_multipliers.get(dlc_level, 1.0)
    
    # 4. 质量加成（AI内容质量评分 0-100）
    quality_score = ai_quality_score(content_type, platforms)
    quality_bonus = int(quality_score * 2)  # 最高200 DVC
    
    # 5. 总奖励计算
    subtotal = base_reward * platform_bonus_rate * dlc_multiplier
    total_reward = int(subtotal) + quality_bonus
    
    # 6. 封顶校验（防止异常）
    max_reward = dlc_level * 200  # L9封顶1800
    total_reward = min(total_reward, max_reward)
    
    return {
        "base_reward": base_reward,
        "platform_bonus": int(base_reward * (platform_bonus_rate - 1)),
        "platform_count": platform_count,
        "dlc_multiplier": dlc_multiplier,
        "dlc_level": dlc_level,
        "quality_bonus": quality_bonus,
        "quality_score": quality_score,
        "total_reward": total_reward,
        "breakdown": {
            "基础奖励": f"{base_reward} DVC",
            "平台加成": f"x{platform_bonus_rate:.1f} ({platform_count}个平台)",
            "DLC加成": f"x{dlc_multiplier:.1f} (L{dlc_level})",
            "质量加成": f"+{quality_bonus} DVC (评分{quality_score})",
            "总计": f"{total_reward} DVC"
        }
    }
```
